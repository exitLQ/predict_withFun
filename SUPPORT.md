# Support

Before asking for help:

1. Read `README.md`, especially Configuration and Troubleshooting.
2. Check `/api/health` and `/api/ready`.
3. Reproduce with demo mode when possible.
4. Search existing GitHub issues.

Use a bug report for reproducible defects and a feature request for proposed
behavior. Include versions, operating system, relevant non-sensitive
configuration, reproduction steps, and sanitized logs.

This community project provides best-effort support. It does not offer service
level guarantees, financial advice, provider billing support, or support for
third-party Polymarket, OpenAI, xAI, Anthropic, Render, Redis, PostgreSQL, or
Sentry accounts.

Report security issues privately according to `SECURITY.md`.
